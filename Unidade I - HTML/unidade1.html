# Unidade 1 - Introdução à criação de páginas web

Visão geral do desenvolvimento web

O desenvolvimento de uma aplicação web envolve várias etapas e aspectos, desde a concepção da ideia até o lançamento final do produto.

Aqui está uma visão geral de como funciona o processo de desenvolvimento de uma aplicação web:

1. Planejamento e análise:
A primeira etapa envolve a definição clara dos objetivos da aplicação web, a identificação do público-alvo e a análise das necessidades dos usuários. Nessa fase, também são realizadas pesquisas de mercado, análise da concorrência e a definição dos recursos e funcionalidades principais da aplicação.

2. Design e prototipagem:
Com base nas informações coletadas na fase de planejamento, começa-se a criar o design da aplicação web. Isso inclui a definição da arquitetura de informações, a criação de wireframes (esboços estruturais da interface) e a criação de protótipos interativos para visualizar a aparência e a usabilidade da aplicação.

3. Desenvolvimento do front-end:
Nesta etapa, os desenvolvedores trabalham na implementação do design da interface do usuário. Utilizando linguagens de marcação como HTML e estilização com CSS, são criadas as páginas web e elementos visuais da aplicação. Também é comum o uso de frameworks e bibliotecas como React, Angular ou Vue.js para agilizar o desenvolvimento.

4. Desenvolvimento do back-end:
A parte do back-end lida com a lógica e o processamento dos dados da aplicação. Aqui, são utilizadas linguagens de programação como Python, Java, Ruby ou Node.js, e frameworks como Django, Ruby on Rails ou Express.js para criar servidores, bancos de dados e APIs (interfaces de programação de aplicativos).

5. Banco de dados:
Em muitas aplicações web, é necessário armazenar e gerenciar dados. Isso é feito usando sistemas de gerenciamento de banco de dados, como MySQL, PostgreSQL ou MongoDB. Os dados são organizados em tabelas ou coleções, permitindo o armazenamento e recuperação eficiente das informações.

6. Testes:
É fundamental realizar testes de qualidade para garantir que a aplicação funcione corretamente e atenda aos requisitos. Testes automatizados e manuais são realizados para verificar se todos os recursos estão operando como esperado, identificar erros e garantir a usabilidade adequada em diferentes dispositivos e navegadores.

7. Implantação e hospedagem:
Uma vez que a aplicação web esteja pronta, ela precisa ser implantada em um servidor para que os usuários possam acessá-la. Existem diversas opções de hospedagem, como serviços em nuvem (por exemplo, AWS, Google Cloud, Azure) ou servidores dedicados. A escolha dependerá dos requisitos de escalabilidade, desempenho e segurança da aplicação.

8. Manutenção e atualizações:
Após o lançamento, a aplicação requer manutenção contínua para correção de bugs, implementação de novos recursos e melhorias de desempenho. As necessidades dos usuários podem evoluir ao longo do tempo, então é importante estar atento a feedbacks e realizar atualizações regulares para manter a aplicação relevante e funcional.

É importante ressaltar que o processo de desenvolvimento de uma aplicação web pode variar dependendo das necessidades e da equipe envolvida. Além disso, muitos dos passos mencionados podem ocorrer simultaneamente ou em iterações, especialmente em metodologias ágeis de desenvolvimento, como o Scrum.

Fundamentos do HTML

Os fundamentos de uma página HTML envolvem a estrutura básica e os elementos essenciais para criar uma página web. Aqui estão os principais elementos que compõem os fundamentos de uma página HTML:

  • Declaração do tipo de documento: A declaração <!DOCTYPE html> é o primeiro elemento que aparece em um documento HTML. Ele informa ao navegador que o documento está escrito em HTML5.
  • Elemento <html>: O elemento <html> é o elemento pai de todos os outros elementos em uma página HTML. Ele envolve todo o conteúdo da página.
  • Elemento <head>: O elemento <head> é usado para fornecer informações sobre o documento HTML.
    Ele contém elementos como <title>, <meta>, <link>, <style>, etc. O conteúdo dentro do <head> não é exibido diretamente na página, mas fornece metadados, informações de estilo e referências externas.
  • Elemento <title>: O elemento <title> define o título da página, que é exibido na barra de título do navegador ou na guia do navegador. É um elemento obrigatório em uma página HTML.
  • Elemento <body>: O elemento <body> é usado para conter o conteúdo principal da página, como texto, imagens, links e outros elementos HTML. O conteúdo dentro do é exibido diretamente na página web.
  • Elementos de cabeçalho (<h1> a <h6>): Os elementos de cabeçalho são usados para criar títulos e subtítulos em uma página HTML. O elemento <h1> é o título principal e <h6> é o título menos importante. Eles são usados para estruturar o conteúdo e indicar a hierarquia dos cabeçalhos.
  • Elemento de parágrafo (<p>): O elemento <p> é usado para criar parágrafos de texto. É usado para agrupar blocos de texto relacionados.
  • Elemento de hyperlink (<a>): O elemento é usado para criar um hyperlink, permitindo aos usuários clicar em um texto ou imagem para navegar para outra página ou seção da página atual. O atributo href define o destino do link.
  • Elemento de imagem (<img>): O elemento <img> é usado para exibir uma imagem na página HTML. O atributo src especifica a URL ou o caminho do arquivo da imagem, e o atributo alt fornece um texto alternativo que é exibido caso a imagem não possa ser carregada.
