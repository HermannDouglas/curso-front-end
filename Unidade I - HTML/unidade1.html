# Unidade 1 - Introdução à criação de páginas web

Visão geral do desenvolvimento web

O desenvolvimento de uma aplicação web envolve várias etapas e aspectos, desde a concepção da ideia até o lançamento final do produto.

Aqui está uma visão geral de como funciona o processo de desenvolvimento de uma aplicação web:

1. Planejamento e análise:
A primeira etapa envolve a definição clara dos objetivos da aplicação web, a identificação do público-alvo e a análise das necessidades dos usuários. Nessa fase, também são realizadas pesquisas de mercado, análise da concorrência e a definição dos recursos e funcionalidades principais da aplicação.

2. Design e prototipagem:
Com base nas informações coletadas na fase de planejamento, começa-se a criar o design da aplicação web. Isso inclui a definição da arquitetura de informações, a criação de wireframes (esboços estruturais da interface) e a criação de protótipos interativos para visualizar a aparência e a usabilidade da aplicação.

3. Desenvolvimento do front-end:
Nesta etapa, os desenvolvedores trabalham na implementação do design da interface do usuário. Utilizando linguagens de marcação como HTML e estilização com CSS, são criadas as páginas web e elementos visuais da aplicação. Também é comum o uso de frameworks e bibliotecas como React, Angular ou Vue.js para agilizar o desenvolvimento.

4. Desenvolvimento do back-end:
A parte do back-end lida com a lógica e o processamento dos dados da aplicação. Aqui, são utilizadas linguagens de programação como Python, Java, Ruby ou Node.js, e frameworks como Django, Ruby on Rails ou Express.js para criar servidores, bancos de dados e APIs (interfaces de programação de aplicativos).

5. Banco de dados:
Em muitas aplicações web, é necessário armazenar e gerenciar dados. Isso é feito usando sistemas de gerenciamento de banco de dados, como MySQL, PostgreSQL ou MongoDB. Os dados são organizados em tabelas ou coleções, permitindo o armazenamento e recuperação eficiente das informações.

6. Testes:
É fundamental realizar testes de qualidade para garantir que a aplicação funcione corretamente e atenda aos requisitos. Testes automatizados e manuais são realizados para verificar se todos os recursos estão operando como esperado, identificar erros e garantir a usabilidade adequada em diferentes dispositivos e navegadores.

7. Implantação e hospedagem:
Uma vez que a aplicação web esteja pronta, ela precisa ser implantada em um servidor para que os usuários possam acessá-la. Existem diversas opções de hospedagem, como serviços em nuvem (por exemplo, AWS, Google Cloud, Azure) ou servidores dedicados. A escolha dependerá dos requisitos de escalabilidade, desempenho e segurança da aplicação.

8. Manutenção e atualizações:
Após o lançamento, a aplicação requer manutenção contínua para correção de bugs, implementação de novos recursos e melhorias de desempenho. As necessidades dos usuários podem evoluir ao longo do tempo, então é importante estar atento a feedbacks e realizar atualizações regulares para manter a aplicação relevante e funcional.

É importante ressaltar que o processo de desenvolvimento de uma aplicação web pode variar dependendo das necessidades e da equipe envolvida. Além disso, muitos dos passos mencionados podem ocorrer simultaneamente ou em iterações, especialmente em metodologias ágeis de desenvolvimento, como o Scrum.

O desenvolvimento client-side é baseado em 3 camadas principais: informação, formatação e comportamento.
Informação: Fundamentos do HTML

Os fundamentos de uma página HTML envolvem a estrutura básica e os elementos essenciais para criar uma página web. Aqui estão os principais elementos que compõem os fundamentos de uma página HTML:

  • Declaração do tipo de documento: A declaração <!DOCTYPE html> é o primeiro elemento que aparece em um documento HTML. Ele informa ao navegador que o documento está escrito em HTML5.
  • Elemento <html>: O elemento <html> é o elemento pai de todos os outros elementos em uma página HTML. Ele envolve todo o conteúdo da página.
  • Elemento <head>: O elemento <head> é usado para fornecer informações sobre o documento HTML.
    Ele contém elementos como <title>, <meta>, <link>, <style>, etc. O conteúdo dentro do <head> não é exibido diretamente na página, mas fornece metadados, informações de estilo e referências externas.
  • Elemento <title>: O elemento <title> define o título da página, que é exibido na barra de título do navegador ou na guia do navegador. É um elemento obrigatório em uma página HTML.
  • Elemento <body>: O elemento <body> é usado para conter o conteúdo principal da página, como texto, imagens, links e outros elementos HTML. O conteúdo dentro do é exibido diretamente na página web.
  • Elementos de cabeçalho (<h1> a <h6>): Os elementos de cabeçalho são usados para criar títulos e subtítulos em uma página HTML. O elemento <h1> é o título principal e <h6> é o título menos importante. Eles são usados para estruturar o conteúdo e indicar a hierarquia dos cabeçalhos.
  • Elemento de parágrafo (<p>): O elemento <p> é usado para criar parágrafos de texto. É usado para agrupar blocos de texto relacionados.
  • Elemento de hyperlink (<a>): O elemento é usado para criar um hyperlink, permitindo aos usuários clicar em um texto ou imagem para navegar para outra página ou seção da página atual. O atributo href define o destino do link.
  • Elemento de imagem (<img>): O elemento <img> é usado para exibir uma imagem na página HTML. O atributo src especifica a URL ou o caminho do arquivo da imagem, e o atributo alt fornece um texto alternativo que é exibido caso a imagem não possa ser carregada.

Informação: Tags e elementos HTML

Aqui estão algumas das tags HTML mais utilizadas:

<html>: o início e o fim do documento HTML.
<head>: Contém informações sobre o documento HTML, como título e metadados.
<title>: o título da página exibido na barra de título do navegador.
<body>: Contém o conteúdo visível da página.
<h1> a <h6>: Define os cabeçalhos de diferentes níveis, sendo <h1> o mais importante e <h6> o menos importante.
<p>: Define um parágrafo de texto.
<a>: Cria um link para outra página ou recurso.
<img>: Insere uma imagem na página.
<ul>: Cria uma lista não ordenada.
<ol>: Cria uma lista ordenada.
<li>: Define um item em uma lista.
<div>: Define uma seção genérica ou contêiner.
<span>: Define uma seção genérica dentro de um texto.
<table>: Cria uma tabela.
<tr>: Define uma linha em uma tabela.
<td>: Define uma célula de dados em uma tabela.
<th>: Define um cabeçalho de célula em uma tabela.
<form>: Cria um formulário de entrada de dados.
<input>: Cria um campo de entrada, como um campo de texto ou um botão.
<textarea>:Cria uma área de texto multi-linhas para entrada de texto.
Formatação: Conceitos básicos de CSS

CSS (Cascading Style Sheets) é uma linguagem de estilo utilizada para definir a aparência e o layout de elementos em um documento HTML. A seguir, apresento alguns conceitos básicos de CSS:

  • 1. Seletores: Os seletores são usados para identificar os elementos HTML aos quais você deseja aplicar estilos. Existem vários tipos de seletores, como seletores de elemento (por exemplo, "p" para parágrafos), seletores de classe (por exemplo, ".nome-da-classe"), seletores de ID (por exemplo, "#id-do-elemento"), seletores de atributo e seletores de descendência.

  • 2. Propriedades: As propriedades são utilizadas para definir o estilo de um elemento selecionado. Existem diversas propriedades CSS disponíveis, como color (cor do texto), font-size (tamanho da fonte), background-color (cor de fundo), margin (margem externa), padding (espaçamento interno), entre outras.

  • 3. Valores: As propriedades CSS recebem valores que determinam como o estilo será aplicado. Por exemplo, color: blue; define a cor do texto como azul. Os valores podem ser palavras-chave pré-definidas (como blue, red, transparent) ou valores numéricos (como 10px, 2em).

  • 4. Classes e IDs: Classes e IDs são atributos HTML que podem ser utilizados como seletores CSS para aplicar estilos específicos a elementos. U ma classe é definida usando o atributo class (por exemplo, < div class="nome-da-classe">) e pode ser referenciada no CSS usando .nome-da-classe. Um ID é definido usando o atributo id (por exemplo, <div id="id-do-elemento">) e pode ser referenciado no CSS usando #id-do-elemento. A diferença principal é que um ID deve ser único na página, enquanto uma classe pode ser aplicada a vários elementos.

  • 5. Regras CSS: Uma regra CSS consiste em um seletor seguido por um conjunto de propriedades e valores entre chaves. Por exemplo:

    p { color: blue; font-size: 16px; }

    Nesse exemplo, todas as tags p (parágrafos) terão o texto em azul e tamanho de fonte de 16 pixels.

  • 6. Cascata: A cascata é um dos princípios fundamentais do CSS. Ela define como os estilos são aplicados e resolvidos quando existem várias regras conflitantes. A cascata leva em consideração a especificidade dos seletores, a ordem em que as regras são declaradas e a herança de estilos dos elementos pais.

Formatação: Seletores e propriedades CSS

Com a chegadas das CSS3, aplicar estilos a elementos individualmente se tornou muito mais fácil. Os novos seletores permitem identificar elementos com base em seus atributos, posição em que aparecem no “elemento pai”, estado (habilitado/desabilitado), entre outros.

  • 1. Seletores por atributos Estes seletores permitem aplicar uma determinada formatação a certos elementos com base no valor de um atributo

    • E[atributo="valor"]: elementos do tipo E cujo atributo seja igual a valor.
    • E[atributo^="valor"]: elementos do tipo E cujo valor do atributo comece com a palavra valor.
    • E[atributo$="valor"]: elementos do tipo E cujo valor do atributo termine com a palavra valor.
    • E[atributo*="valor"]: elementos do tipo E cujo valor do atributo contenha a palavra valor.
    Exemplo: div[id="cabecalho"]{ font-size:xx-large; }

  • 2. Seletores por posição: Estes seletores funcionam com base na posição do elemento no documento ou no controle que o contêm (elemento pai). Por exemplo, podemos selecionar o segundo elemento contido dentro de uma div, independente do seu tipo.

    • root: seleciona o elemento raiz do documento, no caso, é a tag HTML.
    • E:nth-child(N): seleciona o elemento do tipo E que seja o N-ésimo filho do seu elemento pai.
    • E:nth-last-child(N): semelhante ao anterior, mas começando a contagem de baixo para cima.
    • E:nth-of-type(N): seleciona o elemento que seja o N-ésimo do seu tipo.
    • E:nth-last-of-type(N): semelhante ao anterior, mas contando de baixo para cima.
    • E:last-child: seleciona o elemento E que seja o último filho do elemento pai.
    • E:first-of-type: seleciona o elemento E que seja o primeiro de seu tipo dentro do elemento pai.
    • E:last -of-type: semelhante ao anterior, mas com o último elemento.
    • E:only-child: seleciona o elemento E que seja o único filho do seu elemento pai.
    • E:only-of-type: seleciona o elemento que seja o único filho deste tipo dentro de seu elemento pai.
    Exemplo: div:nth-child(1){ background:red; }

  • 3. Seletores por estado: Estes, por sua vez, funcionam com base no estado atual do controle (vazio, habilitado, desabilitado etc.). A seguir, cada um é explicado, facilitando o entendimento.

    • E:empty: seleciona o elemento E que esteja vazio, sem nenhum elemento filho.
    • :target: seleciona o objeto alvo de um link interno (aqueles que são precedidos por # e que direcionam o usuário para um elemento na página).
    • E:enabled: seleciona o elemento E que esteja habilitado para o usuário
    • E:disabled: semelhante ao anterior, mas para elementos desabilitados.
    • E:checked: seleciona elementos que estejam marcados (radios e checkboxes)
    • ::selection: formata o conteúdo selecionado pelo usuário.
    • :not(P): seleciona os elementos que não atendem ao seletor simples P.

  • 4. Combinadores: Combina outros seletores para direcionar os elementos em nossos documentos

    Por exemplo: article > p { text-color: red}
