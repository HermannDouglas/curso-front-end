Unidade IV - Angular

O que é o Angular?

Angular é um framework de desenvolvimento de aplicativos web de código aberto mantido pelo Google. Ele é uma plataforma completa para a criação de aplicativos web dinâmicos, permitindo que os desenvolvedores criem interfaces de usuário interativas e responsivas.

O Angular utiliza a linguagem TypeScript, que é uma extensão do JavaScript, para desenvolver aplicativos.

Uma das principais características do Angular é a sua abordagem baseada em componentes onde cada componente é responsável por uma parte específica da interface do usuário. Esses componentes podem ser reutilizados e combinados para criar aplicativos complexos.

O Angular também oferece recursos avançados como:injeção de dependência, testes automatizados, otimização de desempenho e internacionalização.

Ele segue o padrão MVC (Model-View-Controller) e fornece recursos poderosos para o desenvolvimento de aplicativos, como manipulação de eventos, gerenciamento de estado, roteamento, criação de componentes reutilizáveis e comunicação com servidores por meio de chamadas HTTP.

Com o Angular, os desenvolvedores podem criar aplicativos web escaláveis e de alta qualidade, adequados para uma variedade de casos de uso, desde pequenos projetos até grandes sistemas corporativos.
Principais conceitos e fluxos de trabalho do Angular
  1. Componentes: É uma unidade independente que encapsula o comportamento e a aparência. Cada componente possui seu próprio template HTML, estilos CSS e lógica de negócio escrita em TypeScript, e podem ser reutilizados em diferentes partes do aplicativo.
    • Templates: Eles são escritos em HTML com extensões e diretivas específicas do Angular (como *ngFor e *ngIf) para iterar sobre listas ou controlar a exibição condicional de elementos.
    • Data Binding: O Angular oferece recursos avançados de data binding para sincronizar dados entre o componente e o template. Esses mecanismos permitem atualizar e exibir os dados de forma dinâmica na interface do usuário.
  2. Services: São classes que contêm lógica de negócio reutilizável e são injetados nos componentes que precisam deles. Eles são usados para compartilhar dados, realizar chamadas HTTP, gerenciar autenticação, entre outras tarefas.
  3. Injeção de Dependência: Facilita a gestão das dependências entre os diferentes componentes e serviços. Permite receber instâncias de serviços de que necessitam, em vez de criá-los diretamente. Isso torna o código mais modular, testável e flexível.
  4. Roteamento: Com o roteamento, é possível definir rotas para diferentes URLs e associá-las a componentes específicos. Isso permite que os usuários naveguem entre as diferentes partes do aplicativo sem precisar recarregar a página.
  5. Ciclo de vida dos componentes: Os componentes do Angular possuem um ciclo de vida definido, com diferentes fases. Essas fases incluem a inicialização do componente, a atualização dos dados, a detecção de mudanças e a destruição do componente. O desenvolvedor pode aproveitar esses eventos para executar ações específicas em cada fase.
Iniciando o Angular
Requisitos

npm (package manager) https://www.npmjs.com/

Node.js https://nodejs.org

Primeiros passos
  1. Instalando o CLI > sudo npm install -g @angular/cli
  2. Criando os arquivos > ng new my-app
  3. Selecionar a pasta > cd my-app
  4. Executando a aplicação > ng serve --open

Angular: Componentes

Use o Angular CLI (Command Line Interface) para criar um novo componente.
Exemplo, para criar um componente chamado "MeuComponente", execute o seguinte comando no terminal:

ng generate component MeuComponente

Você pode usar o componente em outros componentes ou diretivas do Angular.
Por exemplo, em um arquivo de template HTML de um componente pai:

<app-meu-componente> </app-meu-componente>

Binding

O Angular oferece vários tipos de binding para conectar dados entre o componente e o template
  • Interpolação: Permite que você insira valores de propriedades do componente diretamente no template {{ }}
    <h1>{{ titulo }}</h1>
    <p>{{ contador }}</p>
  • Property Binding (Binding de Propriedade): Permite que você atribua valores de propriedades do componente a atributos de elementos HTML usando a sintaxe de colchetes [ ]
    <input [value]="nome">
    <img [src]="urlImagem">
  • Event Binding (Binding de Evento):Permite que você responda a eventos do usuário, como cliques de botões, usando a sintaxe de parênteses ( ).
    <button (click)="salvar()">Salvar</button>

Também há o Two-Way Binding (Binding Bidirecional), Class Binding (Binding de Classe) e o Style Binding (Binding de Estilo).

Diretivas

As diretivas são as funções que serão executadas sempre que o compilador do Angular as encontrar. Elas aprimoram a capacidade dos elementos do HTML anexando comportamentos personalizados ao DOM.

  • *ngIf: Diretiva estrutural utilizada para renderizar ou não um elemento com base em uma expressão condicional.

    <div *ngIf="mostrarElemento">
       Este elemento será exibido se mostrarElemento for verdadeiro.
    </div>

  • *ngFor: Diretiva estrutural utilizada para iterar sobre uma coleção e renderizar elementos repetidamente.

    <ul>
      <li *ngFor="let item of listaItens">{{ item }}</li>
    </ul>

  • [ngClass] Diretiva de atributo utilizada para adicionar ou remover classes CSS com base em expressões condicionais.

    <div [ngClass]="{'destaque': isDestacado, 'ativo': isActive}">Conteúdo</div>

  • [ngStyle] Diretiva de atributo utilizada para alterar estilos CSS com base em expressões condicionais.

    <div [ngStyle]="{'color': corTexto, 'font-size.px': tamanhoFonte}">Conteúdo</div>

  • (click) Diretiva de evento utilizada para responder a eventos de clique.

    <button (click)="salvar()">Salvar</button>

Angular: Services

Para criar um service ou serviço, utilizamos o seguinte comando no terminal:

ng generate service MeuServico

Isso criará automaticamente os arquivos necessários para o serviço, incluindo um arquivo de classe TypeScript.

O provedor do serviço é definido como 'root' no decorator @Injectable({ providedIn: 'root' }). Isso significa que o serviço será fornecido em nível de aplicativo e estará disponível para todos os componentes.
Agora, podemos injetar e usar o serviço em outros componentes.

O decorator @Injectable indica que a classe é um serviço e pode ser injetada em outros componentes ou serviços.

Em outras palavras: Injeção de dependência é um padrão de projeto no qual uma classe solicita dependências de fontes externas ao invés de criá-las.
