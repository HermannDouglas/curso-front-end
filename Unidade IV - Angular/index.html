Unidade IV - Angular

O que é o Angular?

Angular é um framework de desenvolvimento de aplicativos web de código aberto mantido pelo Google. Ele é uma plataforma completa para a criação de aplicativos web dinâmicos, permitindo que os desenvolvedores criem interfaces de usuário interativas e responsivas.

O Angular utiliza a linguagem TypeScript, que é uma extensão do JavaScript, para desenvolver aplicativos. Ele segue o padrão MVC (Model-View-Controller) e fornece recursos poderosos para o desenvolvimento de aplicativos, como manipulação de eventos, gerenciamento de estado, roteamento, criação de componentes reutilizáveis e comunicação com servidores por meio de chamadas HTTP.

Uma das principais características do Angular é a sua abordagem baseada em componentes. Os aplicativos Angular são construídos a partir de uma hierarquia de componentes interconectados, onde cada componente é responsável por uma parte específica da interface do usuário. Esses componentes podem ser reutilizados e combinados para criar aplicativos complexos.

O Angular também oferece recursos avançados, como injeção de dependência, testes automatizados, otimização de desempenho e internacionalização. Além disso, possui uma ampla comunidade de desenvolvedores que contribuem com bibliotecas e ferramentas adicionais, aumentando ainda mais a sua funcionalidade.
Com o Angular, os desenvolvedores podem criar aplicativos web escaláveis e de alta qualidade, adequados para uma variedade de casos de uso, desde pequenos projetos até grandes sistemas corporativos.
Principais conceitos e fluxos de trabalho do Angular
O Angular é um framework que segue o padrão de arquitetura de software MVC (Model-View-Controller) e utiliza a linguagem TypeScript para o desenvolvimento de aplicativos web. Ele funciona de maneira semelhante a outros frameworks, mas com suas próprias peculiaridades. VVamos aos principais conceitos e fluxos de trabalho do Angular:
  1. Componentes: O Angular é baseado em componentes. Um componente é uma unidade independente que encapsula o comportamento e a aparência de uma parte específica da interface do usuário. Cada componente possui seu próprio template HTML, estilos CSS e lógica de negócio escrita em TypeScript. Os componentes podem ser reutilizados em diferentes partes do aplicativo.
  2. Templates: Os templates definem a estrutura e a aparência dos componentes. Eles são escritos em HTML com extensões e diretivas específicas do Angular. No template, é possível utilizar diretivas estruturais (como *ngFor e *ngIf) para iterar sobre listas ou controlar a exibição condicional de elementos.
  3. Data Binding: O Angular oferece recursos avançados de data binding para sincronizar dados entre o componente e o template. Existem diferentes tipos de data binding disponíveis, como interpolação ({{ }}), property binding ([ ]), event binding (( )), two-way binding ([()]), etc. Esses mecanismos permitem atualizar e exibir os dados de forma dinâmica na interface do usuário.
  4. Serviços: Os serviços são classes que contêm lógica de negócio reutilizável e são injetados nos componentes que precisam deles. Eles são usados para compartilhar dados, realizar chamadas HTTP, gerenciar autenticação, entre outras tarefas. Os serviços ajudam a manter os componentes mais leves e facilitam a reutilização de código.
  5. Injeção de Dependência: O Angular possui um sistema de injeção de dependência embutido, que facilita a gestão das dependências entre os diferentes componentes e serviços. Ele permite que os componentes recebam instâncias de serviços de que necessitam, em vez de criá-los diretamente. Isso torna o código mais modular, testável e flexível.
  6. Roteamento: O Angular possui um módulo de roteamento que permite a criação de aplicativos de várias páginas. Com o roteamento, é possível definir rotas para diferentes URLs e associá-las a componentes específicos. Isso permite que os usuários naveguem entre as diferentes partes do aplicativo sem precisar recarregar a página.
  7. Ciclo de vida dos componentes: Os componentes do Angular possuem um ciclo de vida definido, com diferentes fases. Essas fases incluem a inicialização do componente, a atualização dos dados, a detecção de mudanças e a destruição do componente. O desenvolvedor pode aproveitar esses eventos para executar ações específicas em cada fase.
Iniciando o Angular

Requisitos

Node.js

npm package manager

Primeiros passos

  1. Instalando o CLI > sudo npm install -g @angular/cli
  2. Criando os arquivos > ng new my-app
  3. Selecionar a pasta > cd my-app
  4. Executando a aplicação > ng serve --open
